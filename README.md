# 强化学习机械臂连续控制任务实验报告
**—— 基于 PPO 与 SAC 算法的性能对比与超参配置消融研究**

## 1. 实验背景与任务介绍

### 1.1 代码库与实验摘要
本实验代码库基于深度强化学习框架构建，完整的项目文件目录与实验分支结构如下：

```text
ROBO_MANU/

├── train_advanced.py           # 高阶超参调优与消融实验专属训练脚本
├── plot_metrics.py / scripts   # 实验指标处理与图表生成系列支撑脚本
├── logs/                       # 10万步基础配置下：模型检查点与 npz 评估文件

└── Report.md                   # 本次实验的核心评估报告与图文论证摘要
```

整体框架共包含基础模型与高级调参模型（高阶消融组）的训练模块（如 `train_advanced.py`、`main.py`），并通过 TensorBoard （`tb_logs` 等目录）与监控脚本提取追踪评估结果。实验主要对比并优化了连续控制领域的两大主流算法：**PPO** (Proximal Policy Optimization) 与 **SAC** (Soft Actor-Critic)。
* **PPO 的理论优势**：作为一种同轨（On-policy）策略梯度算法，PPO 通过裁剪代理优势函数（Clipped Surrogate Objective）来严格控制策略更新的步长，有效防止了破坏性的大幅度参数更新。因此 PPO 具有稳健的收敛特性、易部署性以及对绝大多数物理控制任务良好的默认鲁棒性。
* **SAC 的理论优势**：作为一种基于最大熵强化学习的异轨（Off-policy）算法，SAC 的理论核心在于在最大化累积奖励的同时最大化策略的动作熵。这种机制天然鼓励智能体进行更广泛的空间探索，避免过早陷入局部最优。配合经验回放池（Replay Buffer），极大程度地提升了连续动作空间中的样本利用效率。

### 1.2 FetchReach 任务设置与环境配置
本实验评估选定为 OpenAI Gym 机器人系列中的 `FetchReach-v4` 环境。该环境是一个典型的受限连续物理模拟场景，要求控制 Fetch 机械臂的末端夹爪精确移动，触碰并停留于空间内随机生成的红色目标点中。
* **状态空间 (State Space)**：环境的状态反馈基于多层字典（Dict），其主要观测（Observation）包含了机械臂末端夹爪的三维笛卡尔相对坐标 ($x, y, z$)、各主要控制关节的速度和位置变量，以及目标点相对于自身的位置映射信息（共 10 维连续浮点数）。
* **动作空间 (Action Space)**：本任务由于为典型的连续控制，接收的是 4 维连续动作向量 `Box(-1.0, 1.0, (4,))`。其中前三维表征机械臂末端在 $x, y, z$ 三个正交坐标轴向上的目标线性位移；第四维为夹爪开闭系数（但在纯 Touch 触达任务中该指由于不存在抓取行为要求，故默认处于无效或锁定状态）。
* **奖励函数 (Reward Function)**：该环境默认使用经典的稀疏奖励设计（Sparse Reward）。当夹爪未能进入以目标点中心为容避原点的球形阈值范围内时，每一步无效移动就会受到 $-1$ 的时间步固定惩罚。在达到时间步上限或碰到目标点时奖励收敛至 $0$。所以最终模型训练积累的回报整体表现为负值。其绝对值越小、越接近 0，则表明模型规划的到达轨迹用时更短、寻路效率更高。

./miniconda3-latest-linux-Advanced组），并主要针对批次大小（Batch Size）进行了消融实验。

### 4.1 环境交互性能对比（成功率与奖励）
![高阶组评估指标图表](advanced_evaluation_curves.png)
*如图所示（左侧为平均奖励，右侧为成功率）：*
1. **整体跃升**：与基础实验截然不同，在采用高级调参后，PPO 和 SAC 均打破了性能瓶颈，**全部达到了 100% 的稳定成功率**，最终 Reward 逼近 -1.5 左右（极优收敛状态，由于到达目标的惩罚已被降到最低）。这证明了超参数选择在连续控制 RL 任务中具有决定性作用。
2. **Batch Size 消融结论**：
   * **PPO 衍生组**：Baseline 参数的最终收敛回报（-1.52）略微优于 Batch Size=256 组别（-1.84），说明在此配置下默认批次已经获得了较佳的梯度更新方向。
   * **SAC 衍生组**：随着探索参数调优，SAC 表现出了非凡的数据效率，克服了基础组的不收敛问题。无论是 Baseline 还是 Batch Size=512 组别，都能够在极快的时间步内迅速触达并且稳定在 100% 成功率。

## 6. 结论
1. **算法特性与调参需求**：PPO 在该环境下展现出更好的默认鲁棒性，而 SAC 作为 Off-policy 算法则极度依赖超参数调试（特别是探索机制的设定）。
2. **参数为王**：在连续控制任务中，**超参数调优（Hyperparameter Tuning）往往是决定模型能否解题的先决条件**。调参后的 SAC 与 PPO 均发挥出了卓越的上限潜力，成功并且高效地解决了 FetchReach 任务。
